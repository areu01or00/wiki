# Zero-Overhead Introspection for Adaptive Test-Time Compute

## Overview
Zero-Overhead Introspection is the first framework enabling LLMs to anticipate their own success and required computation during generation — without any auxiliary models, tokens, or inference overhead. The core insight is that a small "probing head" (1-3% parameters) can predict task success probability from intermediate hidden states, enabling the model to decide in real-time how much effort to invest, when to make multiple attempts, when to stop, and when to signal success or failure. Unlike Best-of-N which spends fixed budget regardless of marginal benefit, this approach correlates introspection signals with actual success to allocate compute only where it helps.

## Key Contributions
- First zero-overhead introspection framework — no auxiliary models, no extra tokens, no inference latency overhead
- Small probing head (1-3% of parameters) predicts task success probability from hidden states during generation
- Enables adaptive test-time compute: model decides when to stop, retry, or escalate based on its own confidence signals
- Compares against Best-of-N and other fixed-budget methods showing compute allocation is more intelligent than fixed-sweep approaches
- Addresses meta-cognition gap in LLMs: humans use real-time introspection to allocate cognitive effort; LLMs lacked this until now
