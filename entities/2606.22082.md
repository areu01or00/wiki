# CodeTeam: An LLM-Powered Multi-Agent Framework for Repository-Level Code Generation

## Overview
CodeTeam proposes a multi-agent framework where distinct agents (Architect, CTO, Developer, QA) coordinate to generate entire software repositories from natural-language requirements. The architecture separates planning, design selection, implementation, and iterative debugging across agents with bounded context and Git-based coordination.

## Key Facts
- **Authors**: Multiple (see arXiv)
- **Year**: 2026
- **arXiv**: [2606.22082](https://arxiv.org/abs/2606.22082)

## Key Contributions
- Multi-role agent decomposition: Architect agents draft competing software design sketches (SDS), CTO agent evaluates and normalizes into machine-checkable contracts, Developer agents implement under dependency-aware scheduling, QA agent drives iterative repairs
- Retrieval-augmented planning: Design references grounded via NL retrieval
- Bounded-context coordination: Lightweight Git-based cross-file consistency enforcement
- SFT and PE variants both outperform CodeS baselines on SketchEval and NL2Repo-Bench (42.3% pass rate)
