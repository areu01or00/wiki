# MultiHashFormer: Hash-based Generative Language Models

## Overview
MultiHashFormer solves the parameter footprint problem in language models by replacing token embedding matrices with hash-based representations. The key innovation is hash-based autoregression: each token is represented as a unique hash signature (a short sequence of discrete hash IDs generated by multiple independent hash functions). A Hash Encoder compresses the signature into a latent vector for a Transformer decoder, and the Hash Decoder generates the next token's hash signature before mapping it back to text. Evaluated at 100M, 1B, and 3B parameter scales — consistently outperforms standard Transformer LMs and handles multilingual vocabulary expansion with constant parameter footprint.

## Key Contributions
- Hash-based autoregression: first framework to enable many-to-one hash compression in causal/generative LMs
- Hash signature approach: unique short sequence of discrete hash IDs per token via multiple independent hash functions
- Constant-parameter multilingual vocabulary expansion: new languages added without increasing model parameters
- Strong empirical results at 100M, 1B, 3B scales across multiple benchmarks
