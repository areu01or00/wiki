# How Humans, Bots, and Agents Communicate About Vulnerabilities in Pull Requests

## Overview
This paper studies how humans, bots, and AI agents differ in how they communicate about security vulnerabilities in pull request discussions. It analyzes patterns of explicit vulnerability references (CVEs, GHSAs) versus implicit security-related language, revealing systematic differences in how different actors surface and discuss security concerns in collaborative code review.

## Key Facts
- **Authors**: Rooijendijk, Pien; Treude, Christoph; Wessel, Mairieli
- **Year**: 2026
- **arXiv**: [2606.28125](https://arxiv.org/abs/2606.28125)
- **Online Date**: 2026-06-26

## Key Contributions
- Comparative analysis of vulnerability communication patterns across human, bot, and agent actors
- Identification of explicit vs. implicit security language patterns in PR discussions
- Taxonomy of how different stakeholders surface security concerns in collaborative settings
- Implications for AI agents deployed in security-critical code review workflows
