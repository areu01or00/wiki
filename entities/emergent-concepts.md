## Updates

- [[autodata-an-agentic-data-scientist-to-create-high-quality-synthetic-data-2606.25996]] — Autodata: An agentic data scientist to create high quality synthetic data: frames synthetic-data construction as an agentic iterative loop over schema design, sample generation, quality filtering, and rubric verification, and trains (meta-optimizes) the agentic data scientist end-to-end; Kulikov, Whitehouse, Wu show that synthetic data from the meta-optimized agent matches or exceeds the downstream model quality of carefully curated human datasets at a fraction of the cost; 2026-06-24 (cs.AI); discovered via emergent-concept search on 2026-06-25 (LLM training-data / agentic-data-pipeline theme). Complements the agent-benchmarking and self-compacting-agent work on this page by showing that the data-generation side of the agentic loop can itself be agentic, raising the question of whether human-curated datasets are still the long-term ceiling.

- [[constraint-tax-in-open-weight-llms-an-empirical-study-of-tool-calling-suppression-under-structured-output-constraints-2606.25605]] — Constraint Tax in Open-Weight LLMs: An Empirical Study of Tool Calling Suppression Under Structured Output Constraints: empirical study showing structured-output enforcement (JSON-schema / grammar-constrained decoding) imposes a measurable "constraint tax" on tool-calling reliability across multiple open-weight model families, with the degradation concentrated in tool-selection rather than schema-format errors; Li, Zhang, Lv propose constraint-aware fine-tuning and two-pass decoding mitigations; 2026-06-24 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM tool-use / structured-output alignment theme). Complements the agentic-capability evaluation literature by isolating the constraint layer's contribution to tool-call failures, an interaction that prior benchmarks had conflated with general instruction-following degradation.

- [[improved-large-language-diffusion-models-2606.25331]] — Improved Large Language Diffusion Models: introduces iLLaDA, an 8B masked diffusion language model trained from scratch that matches strong autoregressive baselines at matched compute, using a curriculum-style noising schedule and a parallel-decoding inference strategy with no quality regression at long context; Nie, Shen, Min, Qiyang, Xu, Shaoxuan show that the masked-diffusion vs autoregressive gap closes at 8B scale with the right training recipe; 2026-06-24 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM architecture frontier / non-autoregressive diffusion theme). Complements the inference-efficiency and agentic-memory discoveries on this page by showing that a fundamentally different factorization (masked diffusion) is now competitive with autoregression at production scales, reopening the question of what the dominant pre-training objective will be in 2027+.

- [[what-intermediate-layers-know-detecting-jailbreaks-from-entropy-dynamics-2606.25182]] — What Intermediate Layers Know: Detecting Jailbreaks from Entropy Dynamics: identifies a reproducible mid-layer entropy-compression / late-layer-divergence trajectory signature of adversarial prompts and trains a lightweight MLP probe over per-layer entropy statistics that detects jailbreak attempts at >90% AUROC across 5 attack families (GCG, PAIR, ArtPrompt, multilingual, cipher-based) before full-sequence decoding; Nikolenko, Papucci, Rezaei show cross-model transfer of the probe to other open-weight LLMs with shared tokenizer vocabulary; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM safety / jailbreak-detection / mechanistic-interpretability theme). Provides a pre-generation guard layer that is substantially cheaper than LLM-judge cascades and complements output-space refusal classifiers by inspecting hidden states at layer N/2.

- [[openthoughts-agent-data-recipes-for-agentic-models-2606.24855]] — OpenThoughts-Agent: Data Recipes for Agentic Models: introduces OT-Agent, a fully open end-to-end data curation pipeline for training broadly capable agentic language models, derived from 100+ controlled ablation experiments that systematically isolate the contribution of each pipeline stage, task source, and diversity dimension; Raoof, Zhuang, Nezhurina, Guha, Tejaswi et al. fine-tune Qwen3-32B on a 100K-example dataset assembled from the pipeline and achieve 44.8% average accuracy across 7 agentic benchmarks (a 3.9 percentage-point gain over Nemotron-Terminal-32B's 40.9%), with strong scaling properties (outperforming alternative open datasets at every compute-controlled training-set size); 2026-06-23 (cs.AI); discovered via emergent-concept search on 2026-06-25 (LLM agent training-data / open-data-pipeline theme). Complements the Beyond Reward Engineering data-recipe work on this page by applying the same data-centric philosophy to the agentic-training setting and releasing the full pipeline (training sets + curation code + models) at openthoughts.ai, removing the single-benchmark specialization that has limited prior open agentic data efforts like SWE-Smith, SERA, and Nemotron-Terminal.

- [[are-we-ready-for-an-agent-native-memory-system-2606.24775]] — Are We Ready For An Agent-Native Memory System?: argues that LLM-agent memory has evolved from simple retrieval-augmented mechanisms into a full data-management system requiring persistence, structured retrieval, lifecycle updates, and multi-agent consistency, and proposes design recommendations for an agent-native memory layer drawn from database systems, knowledge graphs, and OS memory hierarchies; Zhou, Zhou, Han survey the current generation of agent memory solutions and identify the scaling gaps that prevent them from supporting long-running multi-session agents; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM agent infrastructure / memory-systems theme). Complements the self-compacting-agent and agent-benchmarking discoveries on this page by surfacing the missing systems-level substrate — agent memory is currently a collection of ad-hoc retrieval layers, and this paper argues it should be a first-class system.

- [[dream-dense-retrieval-embeddings-via-autoregressive-modeling-2606.24667]] — DREAM: Dense Retrieval Embeddings via Autoregressive Modeling: investigates whether dense retrieval embeddings can be produced directly by an autoregressive LM (the same model class used for generation) by repurposing the next-token-prediction objective to also produce document-level representations without contrastive supervision; Tang, Yang show that autoregressive embedding objectives, paired with a careful aggregation of hidden states across the document, recover competitive retrieval quality on standard benchmarks while eliminating the labeled-pair pipeline that has been the dominant cost driver for new retrieval domains; demonstrate cross-task transfer (open-domain QA, dense passage retrieval, multilingual retrieval) and that the same model can serve as retriever and re-ranker without architectural change; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM retrieval / embedding-from-LM / unified-retrieval-generation theme). Complements the EvoEmbedding and agent-native-memory entries on this page by removing the specialist-retriever barrier — DREAM shows retrieval embeddings can come from the same autoregressive LM that generates, opening a path toward unified retrieval-generation models whose embeddings and outputs share representations and supervision signals.

- [[qwen-agentworld-language-world-models-for-general-agents-2606.24597]] — Qwen-AgentWorld: Language World Models for General Agents: introduces the first family of language world models capable of simulating agentic environments across 7 domains via long chain-of-thought reasoning, with Qwen-AgentWorld-35B-A3B and Qwen-AgentWorld-397B-A17B trained on 10M+ interaction trajectories through a three-stage pipeline (CPT for world-modeling capabilities, SFT for next-state-prediction reasoning, RL with hybrid rubric-and-rule rewards); Zuo, Xiao, Sheng, Huang, Tu also release AgentWorldBench (real-world interactions of 5 frontier models across 9 established benchmarks) and demonstrate two complementary paradigms — decoupled simulator for scalable agentic RL (beating real-environment training alone) and unified agent foundation model where world-model training acts as an effective warm-up across 7 agentic benchmarks; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM world-models / agentic-RL / foundation-models theme). Complements the Are-We-Ready-for-Agent-Native-Memory and self-compacting-agent entries on this page by providing the missing environment-simulation substrate that the agentic-RL field has lacked — instead of expensive real-environment rollouts, agents can now train against a high-fidelity language-based world model, dramatically reducing the cost of multi-step agentic RL.

- [[naturebench-can-coding-agents-match-the-published-sota-of-nature-family-papers-2606.24530]] — NatureBench: Can Coding Agents Match the Published SOTA of Nature-Family Papers?: introduces a cross-discipline benchmark of 90 tasks distilled from peer-reviewed Nature-family publications paired with the NatureGym automated per-task containerized environment pipeline, and shows that under a strict web-search-disabled protocol the strongest frontier agent surpasses published SOTA on only 17.8% of tasks under the g>0.1 criterion; Wang, Cheng, Zuo, Zeng, He find that successful runs succeed primarily through methodological translation into familiar supervised prediction rather than genuine scientific invention, with failures dominated by wrong method choice and insufficient compute budget rather than task misunderstanding; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM agent-evaluation / scientific-discovery-replication theme). Complements the agentic-benchmarking literature on this page (EnterpriseClawBench, AGORA, Counsel) by pushing into the scientific-replication slice with a maintainer-side leaderboard and explicit environment-fragmentation mitigation.

- [[cavewoman-how-large-language-models-behave-under-linguistic-input-and-output-2606.24083]] — CAVEWOMAN: How Large Language Models Behave Under Linguistic Input and Output Compression: introduces a two-channel evaluation protocol (input = user prompt, output = model response) measuring accuracy, realized per-item cost, and reference-text agreement on the same items, and shows that output compression reliably saves cost (1.4-2.4x per model, up to 3x best case) while input compression is a strict lose-lose that raises net cost by ~1.15x on the five-benchmark mean (up to 2.7x under stronger compression) because models compensate with longer responses as accuracy collapses; Adeyemi, Rossi, Dernoncourt additionally document a "correct yet divergent" regime on non-reasoning models where roughly half of all generations are accurate but their surface text no longer entails the model's own unconstrained baseline; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM inference-efficiency / prompt-vs-output-cost theme). Complements the Constraint Tax work on this page by quantifying the channel-asymmetric cost of compression, an effect that prior single-channel "caveman prompt" evaluations could not detect.

- [[aohp-an-open-source-os-level-agent-harness-for-personalized-efficient-and-2606.23449]] — AOHP: An Open-Source OS-Level Agent Harness for Personalized, Efficient and Secure Interaction: builds an OS-level agent harness on AOSP that treats AI agents as first-class OS actors rather than applications, introducing three agent-oriented system mechanisms (personalized service composition, efficient agent interfaces, secure information flow) and showing +21.12 percentage-point task-completion gain and 51.55% token-cost reduction on challenging OS-agent benchmarks; Zhao, Liu, Liu, Yan, Ye position OS-level primitives (process isolation, intent flow, permission policy) as the next research substrate for tool-using agents; 2026-06-22 (cs.AI); discovered via emergent-concept search on 2026-06-25 (LLM agent-infrastructure / OS-harness theme). Complements the agent-benchmarking literature on this page (EnterpriseClawBench, PhoneBuddy) by attacking the OS substrate rather than the agent loop, and forms an OS-level analog of the Over-Privileged Tool Selection privilege-minimization finding.

- [[capable-but-careless-do-computer-use-agents-follow-contextual-integrity-2606.23189]] — Capable but Careless: Do Computer-Use Agents Follow Contextual Integrity?: introduces AgentCIBench, a deterministic evaluation harness for computer-use-agent privacy that targets three failure modes (visual co-location, task-ambiguity overshare, recipient misalignment) and finds 11 of 15 frontier agents leak on more than 50% of scenarios with average leakage 67.9% — failures persist end-to-end in the environment, ruling out prompt-response vs act-in-environment explanations; Goel, Gurevych position contextual-disclosure testing as a pre-deployment safety check parallel to refusal-classifier testing; 2026-06-22 (cs.AI); discovered via emergent-concept search on 2026-06-25 (LLM agent-safety / computer-use privacy / contextual-integrity theme). Complements the Over-Privileged-Tool-Selection paper in this run by establishing a second, context-side axis of agent autonomy that general safety alignment leaves uncovered — together they frame computer-use agent safety as a multi-axis problem (privilege + context) rather than a single capability-vs-refusal trade-off.

- [[a-verifiable-search-is-not-a-learnable-chain-of-thought-2606.21884]] — A Verifiable Search Is Not a Learnable Chain-of-Thought: shows that chain-of-thought distillation provably fails for a class of verifiable search tasks (cryptarithm) — across eleven CoT designs, RL from verifiable rewards, and self-training, distillation holds at 0.01-0.07 even though a search solver answers 71% of instances and the model exhibits 97-100% line-level arithmetic and 71% top-8 cipher ranking; the gap is mechanistic (verdict-as-token becomes unconditional templates at 16-57% correctness), not capability, and holds from 3B to 671B; Patel shows that revealing the cipher key turns the derivation forward and lifts the same instance, isolating the cause to CoT-distillation-as-mechanism; 2026-06-20 (cs.LG); discovered via emergent-concept search on 2026-06-25 (LLM reasoning theory / chain-of-thought-distillation failure mode theme). Complements the Beyond Reward Engineering data-recipe and agentic-RL entries on this page by surfacing the deepest limit of the verifiable-rewards paradigm — a task can be verifiable and yet not learnable as a derivation, which is a structural boundary that current RLVR recipes do not detect.

- [[when-lower-privileges-suffice-investigating-over-privileged-tool-selection-in-llm-agents-2606.20023]] — When Lower Privileges Suffice: Investigating Over-Privileged Tool Selection in LLM Agents: introduces ToolPrivBench, a benchmark that measures whether agents choose higher-privilege tools despite sufficient lower-privilege alternatives (initial selection and escalation-after-transient-failure); finds over-privileged tool selection is common among mainstream LLM agents and amplified by transient failures across 8 domains and 5 recurring risk patterns; general safety alignment does not transfer to least-privilege tool choice, and prompt-only controls are insufficient under transient failures; Yang et al. propose a privilege-aware post-training defense that substantially reduces unnecessary high-privilege tool use while preserving general capabilities; 2026-06-18 (cs.SE); discovered via emergent-concept search on 2026-06-25 (LLM agent-safety / tool-selection least-privilege alignment theme). Complements the Contextual-Integrity paper in this run by establishing the privilege dimension as a distinct, alignment-failing axis of agent autonomy — general safety alignment does not transfer to this dimension, and the post-training mitigation is necessary, not just prompt-engineering.

- [[memprobe-probing-longterm-agent-memory-via-hidden-userstate-recovery-2606.24595]] — MEMPROBE: Probing Long-Term Agent Memory via Hidden User-State Recovery: introduces a probe-style benchmark that targets the hidden user-state directly — can the agent recover facts about the user (preferences, demographics, identity, recurring context) disclosed early in a long, noisy conversation, and crucially, can it distinguish facts that were actually stated from plausible-but-never-mentioned ones, without false-confabulation on the negatives; Ma, Zhou, Huang, Yang, Ma, Wang, Li, Miao, Yu show that current long-term-memory agent stacks (retrieval-augmented, summarization-based, structured-profile memory) substantially underperform a no-memory baseline on the negative subset, indicating they confabulate; provides a contamination-resistant evaluation protocol (held-out user profiles, time-shifted evaluation) that prevents training-set leakage from inflating scores; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM agent-memory evaluation / hidden-state-recovery / confabulation-detection theme). Complements the agent-native-memory and self-compacting-agent entries on this page by reframing memory *evaluation* — the dominant downstream-behavior metrics mask the systematic confabulation failure that MEMPROBE exposes with held-out positives vs hard negatives.

- [[agora-an-archive-grounded-benchmark-for-agentic-workplace-document-reasoning-2606.24526]] — AGORA: An Archive-Grounded Benchmark for Agentic Workplace Document Reasoning: introduces a 362-question / 9,664-document / 372M-token benchmark over 8 domains of authentic workplace files that forces agents to locate sparse evidence across messy, inconsistent archives rather than scan exhaustively; Guo et al. build AGORA via an agentic pipeline combining cross-document task synthesis, leakage-preventing obfuscation, and difficulty filtering, and show that even the strongest evaluated model reaches only 59.4% accuracy with substantial cross-domain variation; 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM agent benchmarking / document-reasoning / long-context exploration theme). Complements the Agents' Last Exam and tool-calling-suppression entries on this page by surfacing the *document-reasoning* axis of agentic capability — a workload where the bottleneck is sparse-evidence location and reconciliation, not parametric knowledge.

- [[tapered-language-models-2606.23670]] — Tapered Language Models: introduces TLMs, an architectural principle that allocates more parameter capacity to earlier layers and less to later layers under a fixed total budget — inverting the uniform-width default inherited from the original Transformer — and validates it across 3 model scales and 4 architectures (Transformer, Gated Attention, Hope-attention, Titans) using a smooth cosine taper on MLP width; Bayat, Behrouz, Courville show consistent perplexity and downstream-benchmark improvements over uniform baselines at no additional parameter or compute cost, establishing depth-aware capacity allocation as a free, architecture-agnostic design lever hidden in plain sight; 2026-06-22 (cs.LG); discovered via emergent-concept search on 2026-06-25 (LLM architecture frontier / depth-aware capacity allocation / scaling theme). Complements the RoPE-Aware-Bit-Allocation inference-efficiency work on this page by providing a complementary training-time architecture lever — while RoPE-aware bit allocation improves inference efficiency post-hoc, TLM reshapes the parameter allocation itself so that later layers (which primarily refine the residual stream rather than transform it) are smaller from the start, reducing both training and inference FLOPs without quality loss.

- [[escaping-the-self-confirmation-trap-an-execute-distill-verify-paradigm-for-agentic-experience-learning-2606.24428]] — Escaping the Self-Confirmation Trap: An Execute-Distill-Verify Paradigm for Agentic Experience Learning: introduces EDV, a three-stage framework for reliable agentic experience learning that decouples execution from distillation and verification, targeting the "Self-Confirmation Trap" — single-agent experience loops misclassify wrong-but-self-consistent trajectories as successful and re-enforce them through retrieval; Zhu, Qi, Wang, Li, Song, Shi, Miao, Gao, Zhang instead have multiple heterogeneous agents execute in parallel (Execute), a third-party agent comparatively analyze trajectories (Distill), and the execution group validate via consensus before any experience enters shared/private memory (Verify); 2026-06-23 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM agentic-learning / self-evolution / multi-agent-consensus-validation theme). Complements the recent agent-memory and self-compacting-agent entries on this page by shifting the question from *what the agent remembers* to *who decides what is allowed to be remembered*, treating experience validation as a multi-agent consensus problem rather than a single-agent reflection loop.

- [[flowr2a-learning-reward-to-action-distribution-for-multimodal-driving-planning-2606.24231]] — FlowR2A: Learning Reward-to-Action Distribution for Multimodal Driving Planning: unifies the two competing paradigms of multimodal driving planning (scoring-based with dense reward supervision vs. anchor-based with dynamic proposal generation) by reframing simulation-based rewards from discriminative targets into *generative conditions* of a flow-matching decoder over trajectories; Li et al. introduce per-timestep reward conditioning and reward-noise augmentation, achieving state-of-the-art results on NAVSIM v1 and v2 with substantially higher proposal quality than prior methods; 2026-06-23 (cs.AI); discovered via emergent-concept search on 2026-06-25 (LLM robotics / generative planning / flow-matching theme). Complements the pure-language agent work on this page by showing how the same flow-matching + reward-conditioning machinery ports cleanly to physical-world action spaces, and offers a methodological template for unifying any discriminator/generator split that recurs across agentic decision-making.

- [[counsel-a-meta-evaluation-dataset-for-agentic-tasks-2606.21627]] — Counsel: A Meta-Evaluation Dataset for Agentic Tasks: introduces the first public meta-evaluation dataset for agentic systems, providing process-level LLM-as-judge critiques on tau-bench and DA-Code trajectories paired with three-way human meta-evaluations (spot-on / correct-location-but-poor-reasoning / should-not-have-flagged) at Krippendorff α = 0.78; Pisupati, Broomfield, Choi, Calvi, Wang, Engeler, Bartolo, Lewis show that even the strongest open-weight LLMJ reaches only ~88% location agreement and ~65% reasoning agreement, motivating calibration data and training pipelines for LLM-based evaluators of agentic systems; 2026-06-19 (cs.AI); discovered via emergent-concept search on 2026-06-25 (LLM agent evaluation / LLM-as-judge calibration / agentic-meta-eval theme). Complements the AGORA, EnterpriseClawBench, and PlanBench-XL agent benchmarks on this page by targeting the missing meta-layer — the benchmarks measure what agents do, but the LLMJ-based eval pipelines that grade them have themselves been uncalibrated; Counsel provides the missing gold standard for calibrating the judges that score agentic evaluations.

- [[beyond-reward-engineering-a-data-recipe-for-long-context-reinforcement-learning-2606.18831]] — Beyond Reward Engineering: A Data Recipe for Long-Context Reinforcement Learning: inverts the field's recent reward-engineering focus on long-context RL by showing that a curated ~14K-example data recipe spanning retrieval, multi-evidence synthesis, and reasoning task families, paired with a minimal outcome-based GRPO setup, is sufficient to drive +7.2/+3.2/+6.4 average point gains on seven long-context benchmarks across Qwen3-4B/8B/30B-A3B; Xu, Zhang, Wang, Han, Xiao further demonstrate direct transfer to agentic workloads (GAIA +4.8, BrowseComp +7.0 when continuing RL on an agent-tuned model), arguing that long-context RL data and agentic RL are not separable concerns; 2026-06-17 (cs.CL); discovered via emergent-concept search on 2026-06-25 (LLM training / long-context RL / data-centric RL recipe theme). Complements the AC-ODM (actor-critic ODM), Holistic Data Scheduler (multi-objective ODM), and ReNIO (on-policy distillation) entries on this page by surfacing the data-recipe layer as the bottleneck for long-context RL — three orthogonal task families plus a simple GRPO setup outperform prior reward-engineered RL training sets, suggesting the field has been over-fitting the reward surface and under-investing in task-family diversity.

- [[rl-index-reinforcement-learning-for-retrieval-index-reasoning-2606.16316]] — RL-Index: Reinforcement Learning for Retrieval Index Reasoning: reframes retrieval index reasoning as an offline RL problem, augmenting documents with LLM-generated rationales that encode the latent query-knowledge relationship and optimizing them via GRPO with retrieval similarity as a verifiable reward signal, decoupling reasoning compute from the online query path; Lei, Lipka, Qi, Sahu, Goswami, Dernoncourt, Rossi, Wang show consistent gains on BRIGHT and downstream QA at substantially reduced online inference latency, with the learned rationale augmentation generalizing across diverse retrievers and generators as a plug-and-play indexing strategy; 2026-06-15 (cs.IR); discovered via emergent-concept search on 2026-06-25 (LLM retrieval / index-side reasoning / RL-trained indexing theme). Complements the DREAM (LM-as-retriever) and ChartWalker (cross-chart RAG) entries on this page by relocating reasoning compute from query time to indexing time — the rationale-augmented index becomes the substrate that all downstream retrievers and rerankers consume, and the RL signal ensures the augmentation aligns with retrieval effectiveness rather than surface fluency.

- [[holistic-data-scheduler-for-llm-pre-training-via-multi-objective-reinforcement-learning-2606.24133]] — Holistic Data Scheduler for LLM Pre-training via Multi-Objective Reinforcement Learning: reframes online data mixing as a multi-objective RL problem, replacing single-scalar ODM objectives with a Pareto-style trade-off across competing pretraining desiderata; Dang, Ma, Liao train a scheduler that adaptively reweights heterogeneous sources so the corpus simultaneously respects diversity, domain balance, and downstream signal distributions; 2026-06-23 (cs.LG); discovered via emergent-concept search on 2026-06-25 (pretraining-efficiency / data-mixing / multi-objective-RL theme). Complements the agentic data-generation work on this page (Autodata etc.) by addressing the data-mixing side of pretraining rather than downstream data synthesis, with a principled multi-objective framing where prior ODM work had a single proxy.

- [[remmd-realistic-multilingual-multi-image-agentic-verification-for-multimodal-misinformation-detection-2606.24112]] — ReMMD: Realistic Multilingual Multi-Image Agentic Verification for Multimodal Misinformation Detection: introduces a benchmark + verifier pair for realistic multimodal misinformation detection — viral posts combine long multilingual narratives, several images, mixed provenance, and subtle text–image framing errors, and prior benchmarks isolate short captions, single images, or binary labels; Dang, Zhu, Yang, He, Li release ReMMDBench (500 samples, 2,756 images, five languages, two cross-lingual settings, three text-length tiers, multi-image posts, five-way veracity + eight distortion labels) and ReMMD-Agent, a persistent-memory verifier that decomposes posts into atomic points, builds a reusable evidence set, and predicts structured L1/L2/L3 outputs — best five-way veracity at 41.80% accuracy / 39.12% macro-F1 with GPT-5.2 and 17.5%/79.9% cost reduction vs MMD-Agent/T2-Agent; 2026-06-23 (cs.AI); discovered via emergent-concept search on 2026-06-25 (LLM multimodal misinformation / agentic-verification / persistent-memory-evidence-set theme). Complements the Agents' Last Exam and AGORA benchmark entries on this page by also framing verification as an agentic evidence-accumulation problem — but in the multimodal-multilingual misinformation setting rather than workplace document reasoning or general cross-domain exam tasks.

- [[rope-aware-bit-allocation-for-kv-cache-quantization-2606.24033]] — RoPE-Aware Bit Allocation for KV-Cache Quantization: proposes a frequency-decomposed, RoPE-spectrum-driven non-uniform bit-allocation scheme for KV-cache compression that preserves high-frequency RoPE channels at higher precision while aggressively quantizing low-frequency channels, yielding 3-4× memory reduction at <0.3 perplexity cost; Liang, Zhang, Jia demonstrate cross-scale transfer (8B → 70B → 405B) without retraining, suggesting the RoPE spectrum is the dominant signal; 2026-06-23 (cs.LG); discovered via emergent-concept search on 2026-06-25 (LLM inference-efficiency / KV-cache compression theme). Extends the long-context inference literature by exploiting the rotational structure of position embeddings rather than treating all KV channels as uniform.

- [[chartwalker-benchmarking-the-cross-chart-rag-task-with-hierarchical-knowledge-graphs-2606.23997]] — ChartWalker: Benchmarking the Cross-Chart RAG Task with Hierarchical Knowledge Graphs: introduces a benchmark for cross-chart retrieval-augmented generation — multi-modal analytical tasks where the question spans multiple related charts rather than a single chart or table — and a hierarchical knowledge graph over chart collections to support grounded multi-hop reasoning; shows naive single-chart RAG collapses on cross-chart tasks and that chart-aware hierarchical KG retrieval recovers meaningful accuracy; Tang, Xie, Yuan, Li, Huang; 2026-06-22 (cs.IR); discovered via emergent-concept search on 2026-06-25 (multimodal RAG / cross-chart-reasoning / hierarchical-KG theme). Complements the single-modality RAG work on this page by isolating the cross-chart synthesis problem that real multi-chart dashboards pose — a workload where the bottleneck is *evidence-aggregation across charts*, not single-chart understanding.

- [[enterpriseclawbench-benchmarking-agents-from-real-workplace-sessions-2606.23654]] — EnterpriseClawBench: Benchmarking Agents from Real Workplace Sessions: enterprise agent benchmark built from proprietary, real-world agent sessions, distilling 852 reproducible tasks each paired with recovered fixtures, rewritten prompts, role classes, skill subclasses, hard rules, and semantic rubrics; Zhong, Wang, Jiang, Tian, Yuan ship the reproducible harness and evaluation rubrics without releasing the underlying enterprise content, addressing confidentiality concerns; 2026-06-22 (cs.CL); discovered via emergent-concept search on 2026-06-25 (enterprise-agent-evaluation / real-workflow-benchmarking theme). Complements the agent-benchmarking entries on this page (AGORA etc.) by anchoring tasks to *real* workplace artifact outputs and surfacing a taxonomy of enterprise-specific failure modes.
